Architecture, Specs & Standards
01 — SAP-1 Architecture
Simple As Possible

The SAP-1 (Simple As Possible 1) is a minimal 8-bit computer architecture described by Malvino and Brown in Digital Computer Electronics and popularized by Ben Eater's YouTube series. It represents the simplest complete stored-program computer capable of executing a meaningful instruction set.

The architecture features an 8-bit data bus, a 4-bit address space (16 bytes of RAM), a basic ALU supporting addition and subtraction, and a control unit driven by a microcode ROM. All registers communicate over a shared bus, with control signals gating data onto and off the bus each clock cycle.

The fetch-decode-execute cycle operates in discrete steps called T-states. Each instruction is broken into microoperations — typically 2 fetch steps and 1–3 execute steps — with the control ROM outputting the appropriate control word for each T-state based on the current opcode.

Data Bus

8-bit shared bus. All registers tri-state onto the bus. Only one source drives the bus at any time — enforced by control signals.

Address Space

4-bit program counter and MAR give 16 addressable locations. 16 bytes of RAM is sufficient for demonstration programs.

Control Unit

Microcode stored in AT28C16 EEPROMs. Each address encodes opcode + T-state; output is a 16-bit control word enabling/disabling subsystem actions.

Clock

Variable frequency from sub-1Hz (step-through debug) to several kHz. 555-timer based, switchable between manual single-step and automatic modes.

02 — STM32 Visualizer
Novel Contribution

The team's primary contribution beyond the base SAP-1 design is a real-time bus visualization system built around an STM32F411 (Black Pill) microcontroller. The system monitors 71 internal signals — data bus lines, address lines, control word bits, register outputs, and ALU flags — and displays their state in real time on a 4" TFT LCD over SPI.

Because the STM32 has limited GPIO, signal acquisition uses nine daisy-chained 74HC165 parallel-in serial-out shift registers, each capturing 8 signals. The STM32 clocks out all 72 bits (71 signals + 1 spare) in a single SPI burst per clock cycle, allowing non-intrusive observation of the entire bus state without loading the TTL logic.

04 — Codes & Standards
Applicable References
74LS / 74HC
Texas Instruments TTL and CMOS logic family datasheets. All ICs operate within specified VCC (5V ±5%), IOH/IOL fan-out, and propagation delay parameters.
AT28C16
Microchip AT28C16 EEPROM datasheet — 2K×8 CMOS EEPROM. Programming via T48 programmer with verified write/verify cycle.
SPI Protocol
Serial Peripheral Interface — standard 4-wire synchronous protocol (MOSI, MISO, SCK, CS) used between STM32 and shift register chain and between STM32 and TFT display.
STM32 HAL
STMicroelectronics Hardware Abstraction Layer (HAL) used for all peripheral drivers. Firmware developed in STM32CubeIDE.
Malvino & Brown
Digital Computer Electronics, 3rd Ed. — Primary architectural reference for SAP-1 design, instruction set, and microcode structure.
Ben Eater
eater.net 8-bit breadboard computer series — implementation reference for all base SAP-1 subsystems including clock, ALU, registers, RAM, and output stage.
