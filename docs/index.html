01 — Problem Statement
The Challenge

Computer architecture is taught almost entirely through simulation and abstraction. Students interact with CPUs through software models and HDL simulators — tools that, while useful, obscure the physical reality of how a processor works. The gap between understanding a block diagram and understanding why a control signal pulse causes a register to latch a value is significant, and most curricula never close it.

There is no hands-on hardware platform in the current UTC EE curriculum that allows students to build and observe a working CPU at the gate level. Furthermore, existing open-source SAP-1 implementations provide no mechanism to observe real-time internal bus activity during execution — making it difficult to verify correct operation or diagnose faults without external test equipment.

This project addresses both problems: by constructing a working 8-bit computer from discrete 74LS TTL logic, every architectural concept becomes physically tangible. The addition of an STM32-driven visualization system monitoring 71 internal signals simultaneously turns the computer into its own debugging instrument.

02 — Executive Summary
Project Overview
The 8-Bit Breadboard Computer is a fully functional stored-program computer constructed from discrete 74LS-series TTL logic ICs on breadboards, based on Ben Eater's SAP-1 architecture and supervised by Dr. Abdul Ofoli at the University of Tennessee at Chattanooga. The team's primary contribution beyond the base SAP-1 design is a real-time bus visualization system built around an STM32 microcontroller, monitoring 71 internal signals simultaneously via nine daisy-chained 74HC165 shift registers over SPI. The computer executes a complete fetch-decode-execute cycle driven by a variable-frequency clock, with all subsystems — ALU, registers, program counter, RAM, and output stage — implemented in physical hardware across three team members.
Subsystem A

Clock & Arithmetic

Variable-frequency clock module driving the fetch-decode-execute cycle. ALU implemented with 74LS283 adders supporting ADD and SUB with carry/borrow flags.

Lead — Braylon Nix
Subsystem B

RAM & Program Counter

16-byte static RAM with manual programming capability. 4-bit program counter with load, increment, and reset. Physical enclosure and board integration.

Lead — Andrew Adams
Subsystem C

Control Logic & STM32 Visualizer

EEPROM-based microcode defines instruction behavior via control word outputs. STM32 firmware reads 71 bus signals in real time via a 74HC165 shift register chain over SPI.

Lead — Philip Segraves
