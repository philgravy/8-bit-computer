01 — Problem Statement
The Challenge

Computer architecture is almost always taught through simulation and abstraction — HDL simulators and software models that obscure the physical reality of how a processor actually works. Most curricula never close the gap between a block diagram and why a control signal pulse causes a register to latch a value.

This project addresses that directly. By building a working 8-bit computer from discrete 74LS TTL logic, every architectural concept becomes physically observable. The addition of an STM32-driven visualization system monitoring 71 internal signals simultaneously turns the machine into its own debugging instrument — no external logic analyzer required.

02 — Executive Summary
Project Overview
A fully functional stored-program computer built from discrete 74LS TTL logic ICs on breadboards, following Ben Eater's SAP-1 architecture under the supervision of Dr. Abdul Ofoli at UTC. The team's primary contribution is a real-time bus visualization system — an STM32 microcontroller monitoring 71 internal signals simultaneously via nine daisy-chained 74HC165 shift registers over SPI, displayed live on a 4" TFT LCD. In-Progress Computer Build
Subsystem A

Clock & Arithmetic

Variable-frequency clock module driving the fetch-decode-execute cycle. ALU implemented with 74LS283 adders supporting ADD and SUB with carry/borrow flags.

Lead — Braylon Nix
Subsystem B

RAM & Program Counter

16-byte static RAM with manual programming capability. 4-bit program counter with load, increment, and reset. Physical enclosure and board integration.

Lead — Andrew Adams
Subsystem C

Control Logic & STM32 Visualizer

EEPROM-based microcode defines instruction behavior via control word outputs. STM32 firmware reads 71 bus signals in real time via a 74HC165 shift register chain over SPI.

Lead — Philip Segraves
