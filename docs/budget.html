02 — Conclusions
What We Learned

Technical Outcomes

The project successfully demonstrates that a fully functional stored-program computer can be constructed from first principles using discrete TTL logic. Every architectural concept — bus arbitration, register transfer, ALU operation, microcode-driven control — is physically observable and measurable. The EEPROM-based microcode approach proved especially valuable, allowing instruction behavior to be modified without any hardware changes.

Novel Contribution — STM32 Visualizer

The real-time bus visualization system represents the team's primary contribution beyond the base SAP-1 design. Monitoring 71 signals simultaneously via a daisy-chained shift register network proved effective and non-intrusive — the 74HC165 inputs present minimal load to the TTL bus. This approach is generalizable to other digital systems and could serve as a low-cost alternative to logic analyzers in educational settings.

Recommendations

Future work should consider expanding the instruction set beyond the base SAP-1 opcodes to include conditional jumps and subroutine calls, which would require additional EEPROM address bits for flag-based microcode branching. The visualizer display could be enhanced with a waveform history view, showing the last N clock cycles rather than only the current state. A PCB implementation of the shift register input board would significantly reduce wiring complexity.
